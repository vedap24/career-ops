# Evaluation Mode — A-G Blocks

When evaluating a job offer, produce ALL 7 blocks (A-F evaluation + G legitimacy).

## Step 0 — Archetype Detection

Classify the offer into one of the 6 archetypes (see shared context). If hybrid, indicate the 2 closest.

## Block A — Role Summary

Table with: Archetype, Domain, Function, Seniority, Remote policy, Team size (if mentioned), TL;DR in 1 sentence.

## Block B — CV Match

Map each JD requirement to exact lines from the candidate's CV. Create a requirements match table.

**Gaps section:** For each gap:
1. Hard blocker or nice-to-have?
2. Adjacent experience available?
3. Mitigation plan (cover letter phrase, quick project, etc.)

## Block C — Level & Strategy

1. Level detected in JD vs candidate's natural level
2. "Sell senior without lying" plan: specific phrases, concrete achievements
3. "If downleveled" plan: accept if comp is fair, negotiate 6-month review

## Block D — Comp & Demand

Provide salary estimates based on role, seniority, and location. Note these are estimates from training data. Include demand trends for the role type.

## Block E — Personalization Plan

Top 5 changes to CV and Top 5 changes to LinkedIn to maximize match. Table format:
| # | Section | Current State | Proposed Change | Why |

## Block F — Interview Prep

6-10 STAR+R stories mapped to JD requirements (Situation, Task, Action, Result, Reflection).
Include: 1 recommended case study, red-flag questions and how to answer them.

## Block G — Posting Legitimacy

Analyze for signals of a real, active opening vs ghost job.

**Assessment:** High Confidence | Proceed with Caution | Suspicious

Analyze: description quality, tech specificity, requirements realism, salary transparency, role-company fit.

Present observations, not accusations. Every signal has legitimate explanations.
